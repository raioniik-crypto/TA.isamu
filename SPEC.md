# SPEC: UX改善パッケージ（チャットUI・ページ表示・ホーム画面）

## 機能名

Aimo UX改善 v1 — チャットUI刷新・ページ抽出改善・ホーム画面更新

## ユーザーストーリー

Aimo のユーザーとして、**キャラクターと自然に会話している感覚**を得たい。
なぜなら、現在のチャットパネルは大きすぎてアプリ風になり、キャラと話している実感が薄いから。

Aimo のユーザーとして、**Wikipedia 等の記事を本文だけ読みやすく表示**してほしい。
なぜなら、目次・言語リスト・脚注等のノイズが多く、本文に集中できないから。

Aimo のユーザーとして、**ホーム画面の候補URLで実際に使えるサイトに行きたい**。
なぜなら、現在の候補は実用性が低く、最初の体験が弱いから。

## 受け入れ基準

### Release 1: チャットUI刷新

- [ ] キャラクターから吹き出しが伸びる形式で会話が表示される
- [ ] 常時表示は最新1〜3件のメッセージのみ（コンパクト）
- [ ] 吹き出しの下または近くにコンパクトな入力欄がある
- [ ] 入力欄は常時大きく場所を取らない
- [ ] タップ/クリックで過去の会話履歴を展開表示できる
- [ ] 展開時はスクロール可能な履歴リストが表示される
- [ ] 閉じると再びコンパクト表示に戻る
- [ ] AIの口調が親密度に応じて段階的に変化する：
  - 序盤（attachment < 0.2）: 柔らかく優しい敬語（「〜ですね」「〜ですよ」）
  - 中盤（0.2〜0.5）: 少し砕けた敬語（「〜だね」「〜かも」が混ざる）
  - 後半（0.5〜0.8）: 自然なタメ口寄り（上品さは維持）
  - 高親密度（≥ 0.8）: 温かく親密（崩しすぎない）
- [ ] 既存のリアクション（表情変化・吹き出しメッセージ）が引き続き動作する

### Release 2: ページ表示改善

- [ ] Wikipedia ページの目次（`#toc`, `.toc`）が除去される
- [ ] Wikipedia の言語リンク（`#p-lang`, `.interlanguage-links`）が除去される
- [ ] Wikipedia の脚注セクション（`.reflist`, `.references`）が除去される
- [ ] Wikipedia のカテゴリ（`#catlinks`）が除去される
- [ ] Wikipedia の編集リンク・補助UI（`.mw-editsection` 等）が除去される
- [ ] 全サイト共通: セマンティックコンテンツ内でも `<nav>`, `<aside>`, `<footer>` を除去する
- [ ] 解析結果テキストの行間が `leading-[1.6]` 程度で読みやすくなっている
- [ ] 既存の記事表示・YouTube表示が壊れていない

### Release 3: ホーム画面更新

- [ ] 候補URLが5〜6個の実用的なサイトに更新されている
- [ ] ジャンルが偏りすぎず、調べもの・ニュース・動画・技術系をカバーしている
- [ ] 各候補に適切なアイコンとアクション文言がある
- [ ] 既存のBrowserHome UIレイアウトが維持されている

## エッジケース

### チャットUI
- **長文レスポンス**: AIが長い返答をした場合、吹き出しが画面外にはみ出さないよう最大高さを設定し、内部スクロール
- **連続送信**: 送信中（isSending）は入力を無効化し、重複送信を防ぐ（既存動作を維持）
- **モバイル画面**: 320px幅でも吹き出し＋入力欄が収まること。キーボード表示時にレイアウトが崩れないこと
- **YouTube視聴モード**: CompanionViewer 内でもコンパクトチャットが動作すること
- **リアクション競合**: リアクションメッセージ表示中にユーザーが送信した場合、チャットが優先される

### ページ表示
- **Wikipedia 以外のサイト**: Wikipedia 特別処理が他サイトに影響しない（URL判定でガード）
- **Wikipedia モバイル版** (`ja.m.wikipedia.org`): モバイル版URLも処理対象にする
- **article タグが存在しない Wikipedia**: フォールバックで `#mw-content-text` を使う
- **脚注テキストが本文に含まれるケース**: 脚注マーカー `[1]` 等のテキスト残留は許容（完全除去は過剰）
- **6000文字制限**: 既存のトランケーション動作を維持

### ホーム画面
- **リンク切れ**: 候補URLが将来変更される可能性あり。定数配列で管理し変更容易にする（既存パターン維持）
- **YouTube候補**: YouTube URLは既存のYouTube視聴モードに正しく遷移すること

## 技術的制約

### Gotchas（CLAUDE.md 準拠）
- **Hydration**: チャットUIの新コンポーネントで `window.*` を使う場合は `useHydration()` ガード必須
- **Tailwind v4**: スタイル変更は `globals.css` の `@theme` または直接クラス指定。`tailwind.config.*` 不使用
- **Zustand persist**: chat-store は既に persist 付き。新しいUI状態（展開/折りたたみ）は persist 不要（セッション単位）
- **Framer Motion**: 吹き出しアニメーションは既存の `AnimatePresence` パターンを踏襲
- **5ファイル制限**: 1回の変更で5ファイル以上の同時編集禁止

### 既存互換性
- `ChatPanel`, `ChatInput`, `MessageList` は削除ではなく段階的置換
- `AICharacter.tsx` の既存リアクション機構（reaction-store 経由）は維持
- `CompanionViewer.tsx`（YouTube視聴モード）にも新チャットUIを適用
- `apiGuard()` は外さない
- `buildSystemPrompt()` のシグネチャ変更は API ルート側も追従
